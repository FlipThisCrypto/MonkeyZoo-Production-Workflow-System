# Trait Overuse Guide

Traits are seasoning, not structure. A script should still work if a catchphrase, prop, or tick is removed.

## Overuse Signals

Watch for these warning signs:

- A character says the same catchphrase in multiple unrelated scenes.
- A physical tick appears in every panel instead of only meaningful beats.
- A running prop is mentioned when it does not affect blocking, emotion, or story.
- A weakness makes the character fail the same way every issue.
- A talent lets one character solve problems that should belong to the ensemble.
- A recurring mystery gets new clues but no rest, answer, or emotional payoff.
- A relationship dynamic forces two characters together when the scene needs someone else.

## Suggested Frequency

Use the `recommended_frequency` field as a ceiling, not a requirement.

- `often`: can appear when relevant, but not in every panel.
- `sometimes`: one or two meaningful appearances in an issue.
- `rarely`: special beats only.
- `special circumstances only`: hold for planned continuity.

## Category-Specific Limits

- Catchphrase: once per issue unless repetition is the point.
- Physical tick: one clear beat per scene at most.
- Running prop: only when it changes action, clue logic, or emotion.
- Signature pose: rotate with approved reference art.
- Specific talent: let other characters contribute.
- Specific weakness: pair with agency or learning.
- Recurring mystery: track open loops before adding new ones.

## Retirement

Retire or mark dormant when:

- The trait no longer fits the character.
- It duplicates a stronger trait from another character.
- It became a filler gag.
- It blocks growth.
- It depends on art that is no longer canon.

Retiring a trait is not failure. It keeps the cast alive.
