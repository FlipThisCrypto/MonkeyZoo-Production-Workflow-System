System Inputs

What the workflow ingests to maintain consistency:

  • Rough Monthly Idea (YAML/Markdown draft)
  • Franchise Canon (Master bibles & ledger)
  • Character Specifications (Granular bibles & traits)
  • World Rules (Location, technology, factions)
  • Visual-Style Rules (Staging, fonts, style lock phrase)
  • Approved Character References (Pose-by-pose reference images)
  • Season-Level Story Plans (Macro-arc blueprints)

System Outputs

What the pipeline produces at each stage:

  • Issue Brief & Outline (Validated structure)
  • Continuity Verdict (Canon safety log)
  • Full script (Markdown script file)
  • Machine-Readable Page/Panel Plan (JSON)
  • Art Prompt Pack & Cover Prompt (Seeds, settings)
  • Generated and Selected Panels (Filtered assets)
  • Lettered Pages & CBZ Package (Digital release)
  • Print/Web PDF Layouts (Release exports)
  • CHIP-0015 Metadata & QA Reports (Audited release logs)

Consistency & Safety Guardrails

Strict Human-in-the-Loop Approval

Nothing becomes canon automatically. Proposed continuity changes remain in the ledger as suggestions until approved by the editor.

Append-Only Continuity Ledger

Every story event, character addition, and relation edit is logged to prevent continuity drift and retro-active changes.

Locked Visual Identities

Character bibles contain locked visual rules (e.g. Clever Monkey is the only character with glasses) which are validated programmatically.

Deterministic Reproducibility

Art prompt packs lock seed values, engine recipes, and ComfyUI workflows so every panel render can be fully reproduced.

Getting Started Guide

1. System Requirements

Python 3.8+, Node.js (for Claude Code, optional), and Flask dependencies installed.

pip install flask pyyaml pillow pytest
2. Create New Issue Brief

Drop a rough markdown/YAML idea file into the inbox folder to begin a new issue.

python 00_SYSTEM/scripts/new_issue.py --idea 01_IDEAS_INBOX/2026-07-idea.md
3. Run Validation Gate

Validate the issue folder structure, page plan, and character bibles before generating art.

python 00_SYSTEM/scripts/validate_issue.py --issue 2026-07_Issue_05
4. Build Release Package

Generate final PDFs, digital CBZ packs, and metadata once art generation and lettering are complete.

python 00_SYSTEM/scripts/build_release.py --issue 2026-07_Issue_05

Tooling Integration Matrix

Claude AI Used for script generation, text QA checks, and continuity analysis (Stages 1–4, 7, 9).

Human Editor Approves character names, resolves continuity proposals, and verifies layout quality.

Adapting to Other Properties

While MonkeyZoo serves as the reference implementation, the core architecture is entirely reusable.

You can adapt this pipeline to another comic or manga series by swapping out the configuration modules:

1. Canon Rules Replace world bibles, factions, and continuity ledgers in 00_SYSTEM/.
2. Character Bibles Define new characters, naming statuses, and locked traits in character-bibles/.
3. Image Models Train ComfyUI LoRAs with your own character reference images.
4. Templates Provide custom page dimensions, comic life templates, and balloon fonts.
